Marché financier à risque — Plateforme d'aide à la decision en cours

Statut : Projet personnel privé (code source non public) · développement actif depuis juin 2026

Objectif

Ce projet est une plateforme modulaire conçue pour améliorer la prise de decision sur les marches financiers à risque d'une pratique réactive et émotionnelle en une démarche pilotée par les données et les statistiques.

Le projet répond à un besoin réel et concret : réduire les biais comportementaux émotionnels (panique, FOMO, revenge trading) en s'appuyant sur des indicateurs de marché, de l'osint, du machine learning et l'analyse de l'historique de trading.

Provider de données : flux de marche multi-sources
Segments de marché : actions, indices, ETF et autres actifs à risque
Conformité réglementaire : Le projet s'adapte aux régulations MiCA (UE) et SEC (États-Unis)

Ce projet démontre une architecture backend modulaire à grande échelle, l'intégration de pipelines Machine Learning en production, le traitement de données financières temps réel, et la conception d'interfaces orientées aide à la décision — tout en gérant un produit avec de vrais enjeux (capitaux réels engagés sur ce système).


Stack technique

Backend

  • Framework : Java 21 / Spring Boot / Spring WebFlux
  • Architecture : Monolithe modulaire organisée par domaines métier
    • risk-engine : gestion du risque et dimensionnement des positions
    • psychology-engine : détection de patterns comportementaux
    • trading-journal : journalisation complète des opérations
    • counterfactual : analyse contrefactuelle (que se serait-il passé ?)
    • market-data-aggregator : agrégation données de marché temps réel
  • Persistance : PostgreSQL avec migrations Flyway versionnées

Frontend

  • Framework : Angular (composants standalone, signals, templates de contrôle de flux natifs @if/@for)
  • Communication : Consommation d'API REST du backend
  • Objectif : Interface orientée aide à la décision, dashboard temps réel, visualisations de stratégie

Machine Learning

  • Framework : Python 3 avec FastAPI
  • Pipeline Machine Learning :
    • Régression logistique, MLP (Multi-Layer Perceptron)
    • Random Forest, XGBoost, modèles d'ensemble
    • Détection de retournements de marché avec probabilités calibrées
    • Validation walk-forward (évite l'overfitting en données financières)
  • Monitoring :
    • Détection de data drift (PSI — Population Stability Index)
    • Modélisation de volatilité (GARCH)
    • Identification de régimes de marché cachés (HMM — Hidden Markov Models)

OSINT & Données externes

  • Module Spring Boot indépendant conçu en prévision d'extraction en microservice
  • Agrégation d'événements externes :
    • Réglementation (MiCA, SEC, OCC circulaires)
    • Lancements d'ETF et indicateurs macro-financiers
    • Signaux de sentiment et donnees d'actualité
    • Statut des brokers et plateformes (maintenance, restrictions regionales)

Messagerie asynchrone

  • Broker : RabbitMQ via Spring AMQP
  • Objectif : Découplage entre composants, robustesse du système

Intégrations externes

  • Données de prix : fournisseurs de marche et flux institutionnels
  • Indicateurs de sentiment : fournisseurs specialises multi-actifs
  • Fréquence : Temps réel ou à intervalles réguliers selon la source

Fonctionnalités clés

1. Moteur de gestion du risque (risk-engine)

  • Calcul automatique de taille de position basée sur le risque défini
  • Stop-loss adaptatif basé sur l'ATR (Average True Range)
  • Détection de "chasse au stop" (washouts avant retournement)
  • Garde-fous anti-surtrading (limite quotidienne de transactions, cooldown entre trades)
  • Calcul de ratio risque/rendement pour chaque opportunité

2. Détection statistique et Machine Learning de probabilités de retournement

  • Prédiction de retournements de marché via ensemble de modèles
  • Calibration de probabilités (fiabilité des prédictions)
  • Fourchettes de prix cibles extraites des modèles
  • Périodes de confiance réduite après détection de drift

3. Module de psychologie du trading (psychology-engine)

  • Détection de patterns comportementaux à partir de l'historique réel :
    • Revenge trading (réaction à une perte)
    • Biais horaires (temps de la journée propice au surtrading)
    • Escalade d'engagement (doublement sur perte)
  • Signal anti-panique combinant :
    • Lecture de marché (technicals + Machine Learning)
    • Statistiques historiques (performance passée en situation similaire)
    • Patterns de réaction du trader détectés historiquement

4. Journal de trading complet

  • Enregistrement de chaque ordre : entrée, sortie, risque/rendement réalisé
  • Annotation libre pour chaque trade (contexte, hypothèse, émotion au moment de la décision)
  • Reconciliation avec l'historique de trading (via API)

5. Analyse contrefactuelle

  • "Que se serait-il passé avec une autre décision ?" — simulation d'alternatives exécutables
  • Identification de cas où les signaux Machine Learning auraient changé le résultat
  • Mesure d'alignement entre la logique du trader et celle des modèles

Architecture modulaire

Le backend est conçu en monolithe modulaire, chaque domaine métier ayant ses responsabilités claires :

  • Risk Engine : API REST privée, reçoit une position proposée, retourne les paramètres de risk-on
  • Psychology Engine : Analyse l'historique de trading, détecte les patterns, alerte sur biais
  • Trading Journal : CRUD complet + reconciliation automatique des operations
  • Counterfactual : Engine de simulation d'alternatives
  • Market Data Aggregator : Consomme les feeds externes, enrichit les données locales
  • Machine Learning Service : Appelle le microservice FastAPI, cache les prédictions, gère le drift
  • OSINT Collector : Agrégateur d'événements externes (prévu pour devenir microservice)

Cette séparation par domaine facilite :

  • Le test unitaire de chaque logique métier
  • L'évolution indépendante (ex: remplacer le modèle Machine Learning sans toucher au journal)
  • La scalabilité future (extraction en microservices si besoin)


Conformité réglementaire

Le projet est conçu pour s'adapter à un cadre réglementaire évolutif :

  • Traçabilité : Journal complet des transactions, audit trail, respect des AML/KYC si nécessaire

Compétences démontrées

  • Conception d'une architecture backend modulaire à grande échelle avec Spring Boot
  • Intégration de pipelines Machine Learning en production — validation, drift detection, calibration
  • Traitement de données financières temps réel et gestion d'incohérences (réseau, slippage)
  • Design d'interfaces orientées aide à la décision (tableaux de bord, alertes, visualisations)
  • Gestion de microservices et découplage asynchrone via messaging
  • Agrégation multi-sources de données externes et enrichissement contexte
  • Contrôle qualité avec SonarQube pour suivre la dette technique, les smells et les vulnérabilités
  • Développement d'un produit avec enjeux réels (capital personnel engagé)

Roadmap & évolution prévue

  • Court terme (Juin - Septembre 2026) :
    • Stabilisation du socle : API backend, connecteur de donnees de marche, journal de trading
    • Modèles Machine Learning baseline (régression logistique, Random Forest)
    • Interface Angular minimal mais fonctionnelle (dashboard trades, alerts)
    • Intégration d'un premier contrôle qualité SonarQube sur le socle applicatif
  • Moyen terme :
    • Enrichissement des modèles (XGBoost, ensemble, calibration probabilités)
    • Psychology engine avec détection de patterns comportementaux
    • Renforcement des Quality Gates SonarQube et suivi des régressions
    • Dashboard complet + visualisations avancées
  • Long terme :
    • Extraction de composants en microservices (Machine Learning, OSINT)
    • Ajout d'autres classes d'actifs (actions, indices, derives selon conformité)
    • Événements OSINT intégrés au contexte décisionnel
